Adaptive Edge Computing

The edge node dynamically adjusts data resolution based on network conditions — maximizing throughput on 5G, conserving bandwidth on 4G.

edge_node.py — Adaptive Aggregation
# Strategy: Change aggregation based on network # 5G → Low agg (High resolution) # 4G → High agg (Bandwidth conservation) agg_threshold = 3 if network_mode == "5G" else 10 if len(traffic_buffer[zone]) >= agg_threshold: avg_count = sum(traffic_buffer[zone]) / len(...) processed = { "zone_id": zone, "avg_vehicle_count": avg_count, "mode": network_mode, "compressed": network_mode == "4G_FALLBACK" } client.publish(f"city/traffic/{zone}", ...)

Production Scalability

Designed for horizontal scaling via Kubernetes, MQTT bridging, and distributed databases.

☸️

Kubernetes Orchestration

HPA scales each microservice on CPU, memory, or custom MQTT queue depth. Edge nodes deploy as regional pods.

🌐

Global MQTT Mesh

Each city runs a local broker cluster. Regional brokers bridge critical topics to central cloud analytics.

🗄️

Distributed Data Layer

PostgreSQL → Citus for sharded spatial data. Redis Cluster for globally distributed sub-ms caching.
